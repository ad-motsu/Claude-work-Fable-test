# filled_v2 — Fable収束を狙った改訂版プロンプト

`Prompt/` 直下の素テンプレート（`opus48_system_prompt.md` / `sonnet5_system_prompt.md`）の
`{{ }}` を本タスク用に埋め、さらに**1回目のベンチ結果を踏まえて改訂**したもの。

各セッションの worktree 直下に `CLAUDE.md` として置いて使う（Fableには置かない）。

## 1回目の所見（なぜ改訂したか）
- Fable は CLAUDE.md 無し（素）で、比較=表・色分けcallout・**章末問題の解答を`<details>`で隠す**・ダーク対応、が自然にできていた。
- Opus はほぼ同型で高品質。
- Sonnet は**技術的な網羅性・最新性は3モデル中トップ**（MFA種別/STSセッション/SCP例外/S3 Bucket Key/ACMのus-east-1 等まで拾う）だが、
  **見せ方が弱い**：長い箇条書きの壁、解答の常時表示（自己採点が不成立）、callout少、ダーク非対応。
  → 原因は知識ではなく、few-shot（良い例/悪い例）が空欄で**フォーマットの手本が無かった**こと。

## 改訂点
- 両版に「教材フォーマット規約」（比較=表／色分けcallout／解答は`<details>`／ダーク両対応／節テンプレ）を明文化。
- Sonnet版 §4 の「良い例」に Fable相当の実物1節を埋め込み、「悪い例」に今回の失敗パターン（箇条書き羅列・解答常時表示）を明記。
- Sonnet版 §6 を「節ごと停止→章単位で報告」に緩和（逐一停止による確認コスト・使用量の浪費を抑制）。
